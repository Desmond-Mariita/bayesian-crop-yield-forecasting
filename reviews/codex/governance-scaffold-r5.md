That means a stub can be “graduated” without a dedicated replacement test as long as the overall suite keeps coverage above 90 percent. Recommended change: add a separate graduation manifest or explicit per-name test requirement so CI fails when a name enters `IMPLEMENTED` without a non-contract test.

2. **Minor** - [docs/INVARIANTS.md](/home/manyee/Documents/bayesian-crop-yield-forecasting/docs/INVARIANTS.md#L27-L27) clause LINV-009. “No task is ‘done’ without a saved, timestamped verification report or evidence artifact” is not something `tools/verify.py` can actually observe; the script can write a report, but it cannot know whether a task is done. This is process/review policy, not hard enforcement. Recommended change: reword the enforcement column to say “process/evidence gate” rather than implying CI can prove task completion.

3. **Minor** - [tools/check_notebooks.py](/home/manyee/Documents/bayesian-crop-yield-forecasting/tools/check_notebooks.py#L37-L74). The notebook gate is brittle because it parses the `IMPLEMENTED` assignment out of a specific test file and accepts only a literal `frozenset(...)` shape. Recommended change: move the graduation ledger into a dedicated manifest/module and load that directly instead of AST-parsing a test file.
