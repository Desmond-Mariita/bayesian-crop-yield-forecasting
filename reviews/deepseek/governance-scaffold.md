### 2. (Minor) R‑hat threshold documentation/code inconsistency

**File:** `src/xai/gates.py` lines 69–70; `docs/INVARIANTS.md` LINV-003; `research_log.md` milestone plan.

The documentation (INVARIANTS and research_log) states “R‑hat < 1.01” as the convergence criterion. The code fails the gate only when `r_hat > R_HAT_MAX` (strict greater-than), meaning an R‑hat value of exactly 1.01 **passes**, which contradicts the documented strict inequality. Use‑cases: the Gelman–Rubin diagnostic typically recommends `R‐hat < 1.01`; 1.01 is borderline and often accepted as a pass in practice, but the documentation should match the implementation.

**Recommendation:** Change the documented threshold to “R‑hat ≤ 1.01” or change the code to `r_hat >= R_HAT_MAX`. Also add a test for the boundary case `r_hat=1.01` in `tests/unit/test_xai_gates.py`.

### 3. (Minor) Frozen dataclass with mutable default for `technical_detail`

**File:** `src/xai/cards.py` line 56.

`ExplanationCard.technical_detail` is typed as `Mapping[str, Any]` but has `field(default_factory=dict)`. Because the dataclass is frozen, the field cannot be reassigned, but the dictionary *contents* can still be mutated after construction. This breaks the spirit of immutability and can lead to subtle side‑effects.

**Recommendation:** Use a frozen mapping, e.g. `types.MappingProxyType`, or document that the returned `Mapping` should be treated as read‑only. Since `__post_init__` runs before freezing, a simple change is to wrap the default factory: `field(default_factory=lambda: {})` and then in `__post_init__` overwrite with `object.__setattr__(self, 'technical_detail', MappingProxyType(self.technical_detail))`.

### 4. (Minor) Stub‑discovery engine may miss re‑exported callables

**File:** `tests/unit/test_stub_contracts.py` line 123 (`_collect_cases`).

The discovery filter `getattr(obj, "__module__", None) != module_name` will skip any callable whose `__module__` is different from the top‑level module, e.g., if a function is imported into `__init__.py` from a private sub‑module. The curriculum modules currently avoid this pattern, but a future developer might add re‑exports without realising the contract test will silently ignore them.

**Recommendation:** Add a comment in the test or in `DEVELOPER_GUIDELINES.txt` that all public stub callables must be defined in the module that is listed in `STUB_MODULES`, not re‑exported. Alternatively, extend the discovery to inspect `__all__` if present.
