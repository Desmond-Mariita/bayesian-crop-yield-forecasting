## Findings

1.  **Severity:** Minor
    **File:** `src/data/acquisition.py`, lines 90 and 94.
    **Finding:** The generic helper function `_fetch_json` is used by both `download_nass_yields` and `download_weather_data`, but its error messages are hardcoded to mention "NASS Quick Stats". If a request made on behalf of `download_weather_data` fails (e.g., due to a POWER API outage or invalid JSON response), the resulting `ConnectionError` will be misleadingly attributed to NASS. The unit test `tests/unit/test_weather_acquisition.py::test_fetch_failure_propagates` implicitly confirms this, as it matches on the generic substring `\"request failed\"` while the actual error message contains "NASS".
    **Recommended change:** Generalize the error messages in `_fetch_json` to be source-agnostic. For example:
    
    In `src/data/acquisition.py`:
    
    ```python
    # line 90
    except urllib.error.URLError as exc:
        raise ConnectionError(f"API request failed: {exc}") from exc
    #...
    # line 94
    except json.JSONDecodeError as exc:
        raise ConnectionError(f"API returned invalid JSON: {exc}") from exc
    ```
